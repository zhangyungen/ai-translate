# Change: Add Chrome Translation Extension with Tencent-Style UX and Reading Mode

## Why
当前项目没有可运行代码，且目标能力涉及新插件架构、翻译引擎集成、页面交互、阅读增强模式，属于新增能力与架构落地，需先建立可审查规范与交付路径。

## What Changes
- 新增 Chrome Manifest V3 翻译插件能力，支持任意源语言到任意目标语言，默认优先英文到中文。
- 新增腾讯翻译插件风格交互（弹窗翻译、整页翻译入口、状态切换与语言切换）。
- 整页翻译采用“视口增量翻译”机制：页面滚动到哪里翻译到哪里，减少首屏等待与无效翻译。
  - 对候选段落引入位置+文本键去重，减少动态 DOM 场景下重复展示。
  - 对连续相同译文跳过展示，避免整页模式重复块。
  - 若当前译文与前一条已展示主翻译结果相同，则跳过展示。
  - 若当前译文与当前位置上下文 5 行任一行一致，则跳过展示。
  - 补充页眉/页脚候选扫描策略：覆盖 `header/nav/footer` 与语义 role 容器中的可见文本叶子节点，避免短文本漏翻。
- 复用并抽象腾讯插件 API 调用机制：
  - 请求统一走 `https://transmart.qq.com/api/imt`。
  - `client_key` 采用腾讯插件同源规则：`("tencent_transmart_crx_" + btoa(navigator.userAgent)).slice(0,100)`。
  - 多语言列表采用本地缓存并按 24 小时 TTL 自动刷新。
  - 文本翻译结果支持缓存（内存 + 本地持久化），滚动翻译时优先命中缓存。
- 新增阅读模式：
  - 对原文进行增强规则分段断句（轻量 NLP：句边界识别、缩写/小数保护、长句二次切分、短句合并）。
  - 分段符号固定为 `  |  `。
  - 对低频词组/难词给出预翻译并在下方展示。
  - 鼠标悬停 0.3 秒或选中词/句后，自动在内容下方显示译文。
  - 展示顺序固定为：分割断句 -> 低频/生僻词翻译 -> 整段翻译。
  - 嵌入翻译内容时按类型（断句/术语/整段翻译）分别与上一段展示内容对比，若相同则跳过展示。
  - 主翻译结果（术语/整段）若与前一条已展示主翻译结果一致，则跳过展示。
  - 主翻译结果若与当前位置上下文 5 行任一行一致，则跳过展示。
  - 断句内容与整段翻译内容使用不同背景色，提升可读性与辨识度。
  - 对少于 3 个单词的文本，不展示断句分割结果（仍保留术语/整段翻译流程）。
- 新增模式互斥约束：
  - 阅读模式与整页翻译同一时刻只能开启一种。
  - 当用户切换模式时，系统需先关闭当前模式，再对目标模式重新执行翻译流程。
- popup 需要展示当前开启功能状态（无/整页翻译/阅读模式）。
- 手动点击 popup 的“整页翻译/阅读模式”开关时，源语言与目标语言以 popup 当前选择为准，并传递到内容脚本执行翻译。
- 新增页面级自动语言规则：
  - 来源语言自动适配当前网页语言。
  - 目标语言按规则自动决策：中文->英文，英文->中文，否则浏览器默认语言。
  - 当来源语言与目标语言相同，提示“无需翻译”并跳过翻译流程。
- 新增代码区排除规则：
  - 代码区内容默认不参与整页翻译、阅读模式候选扫描、hover 翻译。
  - 仅在整页翻译或阅读模式开启时，允许用户选中代码区文本执行选中翻译。
- 建立高内聚低耦合分层结构（领域层、应用层、基础设施层、UI 适配层）。
- 所有枚举/常量集中定义，避免魔法值；异常分支输出 `warn` 日志。
- 增加一键打包命令用于产物构建。
